# YouTube -> Discord via GitHub Actions

Автоматически проверяет YouTube RSS-ленты и отправляет уведомления о **всех новых видео** в Discord через webhook.

## Как работает

- Запускается каждые 5 минут через GitHub Actions `schedule`
- Поддерживает **несколько YouTube-каналов** одновременно
- Получает последние 15 видео из RSS-ленды каждого канала
- Находит **все видео**, которые ещё не были опубликованы в Discord
- Отправляет embed-уведомление в Discord на каждое новое видео (от старого к новому)
- Сохраняет историю отправленных видео в `state.json` с датой и временем (MSK)
- Поддерживает **force-posting** любого видео по запросу
- Поддерживает **reset-state**: обновление state.json без постов в Discord

## Формат state.json

```json
{
  "channels": {
    "UCxxxxxxxxxxxxx": {
      "posted_video_ids": ["videoId1", "videoId2", "videoId3"],
      "updated_at": "16.06.2026 22:00:00 MSK"
    },
    "UCyyyyyyyyyyyyyyy": {
      "posted_video_ids": ["videoIdA", "videoIdB"],
      "updated_at": "16.06.2026 21:55:00 MSK"
    }
  },
  "last_checked": "16.06.2026 22:00:00 MSK",
  "last_updated": "16.06.2026 22:00:00 MSK"
}
```

> Хранит список ID уже опубликованных видео (до 50 штук на канал). При первом запуске видео только сохраняются без отправки уведомлений.

## Настройка

### 1. GitHub Secrets

Перейди в `Settings` -> `Secrets and variables` -> `Actions` -> `New repository secret`

| Секрет | Обязателен | Описание |
|--------|------------|----------|
| `DISCORD_WEBHOOK_URL` | Да | URL вебхука Discord |
| `YOUTUBE_CHANNEL_IDS` | Да | ID каналов через запятую (напр. `UCaaa,UCbbb`) |
| `DISCORD_ROLE_ID` | Нет | ID роли для упоминания в уведомлении |
| `BOT_NAME` | Нет | Имя бота в Discord (по умолчанию: `YouTube Notifier`) |
| `BOT_AVATAR_URL` | Нет | URL аватара бота |

### 2. Получение YouTube Channel ID

1. Открой канал на YouTube
2. Нажми правой кнопкой -> `Просмотр кода страницы`
3. Найди `channel_id` или используй сервис [commentpicker.com/youtube-channel-id.php](https://commentpicker.com/youtube-channel-id.php)

### 3. Создание Discord Webhook

1. Открой нужный канал Discord
2. `Настройки канала` -> `Интеграции` -> `Вебхуки` -> `Создать вебхук`
3. Скопируй URL вебхука

## Логика публикации

- При каждом запуске скрипт берёт последние **15 видео** из RSS-ленды
- Сравнивает их с уже отправленными ID из `state.json`
- Все новые видео, которых нет в истории, отправляются в Discord **по порядку** (от старого к новому)
- Если канал выпустил сразу несколько роликов между запусками — все они будут опубликованы
- История хранит последние **50 ID** на канал
- Между несколькими публикациями подряд добавляется небольшая задержка для предотвращения rate limit Discord

## Force-posting

Чтобы принудительно опубликовать последнее видео с канала:

1. Перейди в `Actions` -> `YouTube to Discord`
2. Нажми `Run workflow`
3. В поле `force_channel_id` введи ID нужного канала
4. Запусти workflow

## Reset State

Чтобы обновить `state.json` **без публикации в Discord** (полезно, если добавил новый канал):

1. Перейди в `Actions` -> `YouTube to Discord`
2. Нажми `Run workflow`
3. В поле `reset_state` введи `true`
4. Запусти workflow

Скрипт соберёт последние 15 видео со всех каналов и сохранит их ID в `state.json` без отправки сообщений.

## Первый запуск

При первом запуске скрипт **не отправляет** уведомления — он только сохраняет текущие видео как уже обработанные. Это сделано для того, чтобы не спамить старыми роликами при первой инициализации.
